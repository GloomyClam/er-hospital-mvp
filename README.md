# 응급 이송 병원 추천 MVP

환자 상태, 병원 가용 자원, 중증질환 수용 가능 여부, 거리 정보를 바탕으로 응급 이송 시 연락할 병원 후보를 추천하고 연락 결과를 기록하는 MVP입니다.

현재는 부산 수영구를 기준으로 기능과 사용 흐름을 검증하고 있습니다.

## 주요 기능

- 공공 API 기반 응급실 병원 조회
- 환자 유형별 병원 필터링
- 환자 구분별 중증질환 수용 가능 가산점
- fallback 좌표 기반 거리 계산 및 거리 점수
- 추천 점수와 추천 근거 표시
- 환자 정보 선택 입력 및 활력징후 안내
- 병원 연락 모달과 `tel:` 전화 연결
- 수락, 거절, 응답 없음 결과 기록
- 거절 사유 기록
- 현재 이송 연락 타임라인
- 최종 수락 병원 표시
- 완료 이송 기록 저장 및 상세 보기
- 완료 이송 기록 CSV 내보내기
- 새 이송 시작 시 현재 이송 정보 초기화

## 현재 MVP 범위

- 조회 지역은 부산광역시 수영구를 기본값으로 사용합니다.
- 환자 정보와 이송 기록은 현재 브라우저의 `localStorage`에 저장됩니다.
- 서버 또는 DB에는 이송 기록을 저장하지 않습니다.
- 브라우저나 기기를 바꾸면 기존 기록이 공유되지 않습니다.
- 일부 병원 좌표는 거리 기능 검증을 위한 임시 fallback 좌표입니다.
- 병원 정보는 공공 API 응답을 기준으로 하며 실시간 현장 상황과 다를 수 있습니다.
- 실제 이송을 결정하기 전 반드시 해당 병원에 전화로 수용 가능 여부를 확인해야 합니다.

## 실행 방법

### 1. 준비

- Node.js 18 이상
- 공공데이터포털에서 발급받은 API 인증키

### 2. 패키지 설치

```bash
npm install
```

### 3. 환경변수 설정

`.env.example`을 복사해 프로젝트 루트에 `.env` 파일을 만듭니다.

```bash
cp .env.example .env
```

`.env`의 `DATA_GO_KR_SERVICE_KEY`에 발급받은 인증키를 입력합니다.

```env
DATA_GO_KR_SERVICE_KEY=발급받은_인증키
```

인증키가 소스 저장소에 포함되지 않도록 `.env` 파일은 외부에 공개하지 마세요.

### 4. 서버 실행

개발 중 파일 변경을 자동으로 감지하려면 다음 명령을 사용합니다.

```bash
npm run dev
```

일반 실행은 다음 명령을 사용합니다.

```bash
npm start
```

서버가 실행되면 브라우저에서 [http://localhost:3000](http://localhost:3000)을 엽니다.

## MVP 데모 시나리오

1. 화면에서 **새 이송 시작** 버튼을 누릅니다.
2. 일반 응급, 중증 환자, 수술 가능성 있음, 입원 가능성 있음 중 환자 유형을 선택합니다.
3. 환자 구분에서 흉통/심혈관, 뇌졸중 의심 등 현장 분류값을 선택합니다.
4. 필요한 경우 주증상, 의식 상태, 활력징후와 현장 메모를 입력합니다.
5. 조건을 만족하는 병원과 각 병원의 추천 점수 및 추천 근거를 확인합니다.
6. 우선 연락할 병원의 **연락하기** 버튼을 누릅니다.
7. 전화 확인 후 거절된 경우 거절 사유를 선택하고 **거절**로 기록합니다.
8. 현재 이송 연락 타임라인에 거절 결과가 추가됐는지 확인합니다.
9. 다음 후보 병원의 **연락하기** 버튼을 누릅니다.
10. 병원이 환자를 수용할 수 있다면 **수락**으로 기록합니다.
11. 현재 이송 영역에서 최종 수락 병원을 확인합니다.
12. 완료 이송 기록에서 완료 시간, 환자 정보, 최종 수락 병원과 연락 횟수를 확인합니다.
13. **상세 보기** 버튼으로 해당 이송의 연락 과정을 펼쳐봅니다.
14. **CSV 내보내기** 버튼으로 완료 이송 기록을 검증 자료로 내려받습니다.
